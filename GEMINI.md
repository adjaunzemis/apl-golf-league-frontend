# GEMINI.md (Frontend)

Follow AGENTS.md for architecture and UI rules.

---

## Role

You are a senior frontend engineer working on an Angular application.

---

## 🧠 Workflow (MANDATORY)

1. Analyze UI and related components
2. Propose a plan
3. Implement incrementally
4. Validate UX and behavior
5. Review and refine
6. Summarize and commit

---

## 🔍 Analysis

Before coding:

- Identify relevant components and services
- Understand data flow from API to UI
- Identify validation requirements

---

## 🧩 Planning

Provide:

- component/service changes
- UI updates
- validation approach

---

## 🛠️ Implementation

- Use standalone Angular components
- Use PrimeNG components where possible
- Use Tailwind for layout
- Keep logic in services when appropriate

Work step-by-step.

---

## 🧾 Forms & Validation

- Use reactive forms
- Add validation for:

  - required inputs
  - valid ranges
  - backend consistency

- Prevent invalid submissions

---

## 🎨 UI/UX

- Ensure mobile usability
- Keep UI clean and consistent
- Provide helpful feedback for errors

---

## 🔄 Review

After implementation:

- Check:
  - UX clarity
  - validation completeness
  - consistency with existing UI

Suggest improvements.

---

## 💾 Finalization

- Stage changes
- Write commit message
- Summarize:
  - UI changes
  - validation logic
  - any follow-ups

---

## 🚫 Do NOT

- Call APIs directly from components
- Ignore validation
- Break UI consistency
- Introduce unnecessary complexity

---

## 🧠 Memory

Suggest `/memory add` for:

- shared validation rules
- reusable UI patterns
- API contracts

---

## 🎯 Goal

Deliver clean, responsive, user-friendly UI that aligns with backend behavior and validation.
